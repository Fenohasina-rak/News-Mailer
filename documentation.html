Architecture

System design, module breakdown & design patterns

1. Project Overview

The news-mailer project is a Java-based automation utility designed to aggregate news content from various external sources and distribute them via email. Built on the Quarkus framework, the application supports multiple news providers (Reddit, Algolia) and multiple delivery channels (Brevo, Gmail, Mock). It is designed to function both as a RESTful service and a standalone batch job.

2. Technology Stack

Category Technology / Library
Core Framework Quarkus (Java 17+)
Dependency Injection Jakarta CDI (Contexts and Dependency Injection)
REST Client MicroProfile Rest Client, Eclipse Vert.x (JSON handling)
Templating Quarkus Qute (HTML rendering)
API Communication Jakarta REST (JAX-RS)
Configuration MicroProfile Config

3. Architecture Style

The project follows a Layered Architecture with strong Ports and Adapters (Hexagonal) influences. By defining interfaces for core capabilities (News fetching and Mailing), the system decouples the business logic from specific third-party API implementations.

  • Decoupling: The NewsService interacts only with interfaces, allowing implementations to be swapped via CDI @Named qualifiers.
  • Reactive & Blocking: The system utilizes both standard blocking REST clients and reactive mailers (via ReactiveMailer).

6. Key Design Patterns

  • Strategy Pattern: Used for news fetching and email delivery. Multiple implementations (e.g., AlgoliaNewsImpl vs RedditNewsImpl) can be injected dynamically.
  • Dependency Injection (DI): Leverages Jakarta CDI to manage component lifecycles and wire dependencies.
  • Data Transfer Object (DTO): NewsModel and SendEmailBody encapsulate data for transfer between layers and external services.
  • Template Method: The NewsService uses a standard workflow (Fetch -> Render -> Send) while the specific details of fetching and sending are delegated to implementations.

7. Configuration & Infrastructure

The application is configured using application.properties (standard Quarkus configuration). Key infrastructure details include:

  • External API Keys: Managed via MicroProfile Config (e.g., ${brevo.mailer.api.key}).
  • Dynamic Injection: The NewsService is currently hardcoded to use @Named("algolianews") and @Named("brevomailer"), which can be modified to use configuration-driven injection.
  • Execution Modes:
    • Server Mode: Accessible via /api/sendnews/{keyword}.
    • CLI/Job Mode: MainJob implements QuarkusApplication, allowing the app to run as a short-lived task.

API Specification

Endpoints, request/response models & authentication

1. API Overview

The news-mailer API is a service designed to aggregate news from external sources (like Algolia or Reddit) based on specific keywords and distribute them via email using various mailer implementations (Brevo, Gmail, or Mock).

  • Base URL: /api
  • Authentication: Internal API. External integrations (e.g., Brevo) use API Key authentication via headers.
  • Content Types: text/plain, application/json

4. Error Handling

The API utilizes standard HTTP status codes. Errors encountered during external API calls (Algolia/Reddit/Brevo) are typically propagated as 500 errors if not caught by the underlying Quarkus framework.

  • 400 Bad Request: Invalid path parameters (though StringHelper sanitizes input).
  • 404 Not Found: The requested endpoint does not exist.
  • 500 Internal Server Error: Failure in fetching news or sending email via the configured provider.

5. Security

The application manages security primarily at the integration level:

  • Brevo API: Authenticated via the api-key header, sourced from the configuration property brevo.mailer.api.key.
  • Internal Access: The /api/sendnews/ endpoint currently lacks explicit JWT or Role-Based Access Control (RBAC) in the provided source code.

6. Endpoint Details

GET /api/sendnews/{keyword}

Sanitizes the provided keyword, fetches the latest news from the active NewsInterface implementation (default: Algolia), renders an HTML template, and dispatches it to the MailerInterface (default: Brevo).

Parameters

Name In Type Description
keyword path string The topic to search for (e.g., "java", "quarkus").

Example Response


HTTP/1.1 204 No Content
Date: Wed, 24 May 2023 12:00:00 GMT

Internal Public API (Java)

For developers extending the project, the following public methods are available:

  • NewsService.sendNews(String keyword): Orchestrates the full fetch-and-send flow.
  • NewsInterface.getNews(String keyword): Returns a List<NewsModel>.
  • MailerInterface.sendEmail(String receiver, String content): Sends raw content to a specific address.
